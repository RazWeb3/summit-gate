# 要件定義書 - Summit Gate

## 1. プロジェクト概要
- **名称**: Summit Gate
- **コンセプト**: 山小屋による山小屋のための「公平な分配・決済」プラットフォーム
- **ビジョン**: 富士山の山小屋が、行政や委託業者の「さじ加減」に左右されず、自分たちの活動実績（トイレ利用等）に基づいた報酬を、透明かつ自動的に受け取り、そのまま山の上で決済に活用できる自律的な経済圏を構築する。

## 2. 解決すべき課題（The Problem: 山小屋視点）
- **分配プロセスの不透明性**: 徴収委託業者から送られてくる金額が「本当に正しいのか」を検証する手段が山小屋側にない（エクセル管理の限界）。
- **事務作業の負担**: 忙しいシーズン中に、どのゲートで何人通ったかの集計や、それに基づく請求業務、スタッフへの歩合給計算などの事務作業が煩雑。
- **資金の「死蔵」**: 銀行振込を待っている間、手元の資金（JPYC）があれば山の上で燃料や食料の仕入れ決済に即座に回せるはずの機会損失。

## 3. なぜブロックチェーン/JPYCなのか（山小屋にとっての利点）
銀行振込では実現できない、山小屋主体のメリットは以下の通りです。

| 特徴 | 銀行振込（現状） | Summit Gate (Blockchain/JPYC) |
| :--- | :--- | :--- |
| **検証可能性** | 業者の報告を信じるしかない。 | **コントラクト内の総額と分配率をいつでも監査可能。** |
| **事務コスト** | 集計、請求、振込確認の手間。 | **データ同期と同時に「受取権利」が自動確定。請求書不要。** |
| **利益配分** | オーナーが手動でスタッフや共同経営者に分配。 | **コントラクトによる「自動売上分配（Split）」が可能。** |
| **決済の連続性** | 一度日本円に戻してからの振込。 | **受け取ったJPYCをそのまま仕入れ業者への支払いに利用可能。** |

### C. 「なぜJPYC（ブロックチェーン）なのか」の決定的理由
銀行振込や現金管理ではなく、JPYCを採用する最大の理由は**「運営による資金の私物化・中抜き・滞留を物理的に不可能にするため」**です。

1. **運営の資金非接触（Zero-Touch Distribution）**:
    * 運営主体（Summit Gateプロジェクト）は、自治体や委託業者から集まった環境保全金を、1円たりとも「預かる」ことも「プールの権利」を持つこともありません。
    * 資金は「業者 → JPYC社 → 分配コントラクト → 各山小屋」というルートを辿り、運営はプログラムの「設定（手数料率など）」を行うだけで、動いている現金には一切触れられません。
2. **法的リスクの完全回避**:
    * 運営が資金をプールした場合、「資金移動業」の免許が必要になりますが、本システムでは「自動精算プログラム」を提供しているだけであり、運営は決済の主体ではありません。
    * 自治体や業者が「中立な第三者（スマートコントラクト）」へ直接デポジットすることで、運営側のコンプライアンス負荷をゼロにします。
3. **分配の不可避性（Immutable Distribution）**:
     * 一度コントラクトへ入金された資金は、プログラムされた実績（Points）に基づいて分配されることが数学的に確定します。
     * 「運営の判断で今月の分配を遅らせる」「特定の小屋への支払いを止める」といった恣意的な操作が不可能なため、山小屋は安心してシステムを利用できます。
 4. **システム利用料の透明性と制限（Fee Governance）**:
     * 運営が受け取る手数料（システム利用料）は、入金時にのみ確定・徴収されます。**過去の入金分に対して遡及的に手数料率を変更することは物理的に不可能**です。
     * 手数料率にはスマートコントラクト上で「20%」のハードキャップ（上限）が設定されており、運営による不当な比率引き上げをシステムレベルで防止しています。

## 4. ターゲット
- **利用者**: 富士山登山客（まずは富士宮ルート限定）
- **ステークホルダー**: 
  - **既存徴収委託業者**: 現在行政から徴収業務を請け負っている民間企業。
  - **山小屋運営者**: 実際に登山客を受け入れる各合目の施設。
  - **富士宮市・山岳組合**: 制度の承認・実証実験のパートナー。
  - **資材運搬業者**: JPYC決済のエコシステム参加者。

## 5. 主要要件
### A. 既存システムとの相乗り・連携（6,000円事前購入モデル）
1. **徴収の委託（柔軟な管理主体）**: 登山客は事前予約アプリで「入山管理料（4,000円）」と「環境保全金（2,000円）」の計6,000円を一括決済する。
2. **理想的な資金流動（Split Payment）**: 
    * クレジットカード決済代行会社（PSP）から、業者側の口座に入金された時点で、即座に2つのルートに分離される。
    * **ルート①（4,000円）**: 「県/自治体」の指定口座へ振込（公金として処理）。
    * **ルート②（2,000円）**: 「JPYC社」の銀行口座へ振込（直接発行スキーム）。
3. **アドオン形式**: 既存の入山料徴収アプリや現地窓口システムに、「環境保全金（2,000円）」の徴収機能をアドオン（追加）する。
3. **「リストバンド」の提供廃止**: 
    *   従来のRFIDリストバンド方式を廃止し、**AIカメラによる完全ウォークスルー認証**を採用する。
    *   これにより、リストバンドの製造・配布・回収にかかる物流コストと環境負荷を完全に排除する。
    *   登山者は何も身につける必要がなく、ストレスフリーに施設を利用できる。
4. **分配の自動化（週次バッチ運用）**: 徴収主体（業者または行政）が、一週間分の一括徴収資金をJPYC社の銀行口座へ振り込む（週1回のオペレーション）。入金確認後、JPYCがSummit Gateのプールに直接発行され、そこから山小屋へ自動分配される。

### B. JPYCを活用した金融・分配システム
1. **プール形成**: 環境保全金の一部（1,800円/人）をスマートコントラクト上のプールに集約。
2. **自律的・即時的な権利確定（Non-custodial Settlement）**:
    *   コントラクトは「資金を一時的に預かるプール」ではなく、入金と同時に各山小屋への「権利（残高）」を数学的に確定させる**自律的な精算ブリッジ**として機能する。
    *   運営は分配ロジックやタイミングを操作できず、プログラムが実績（RFID通過データ）に基づき即座に権利を確定させる。
    *   この設計により、運営側が「他人の資金を管理（カストディ）」しているとみなされる法的リスクを徹底的に排除する。
3. **ダイレクト分配・非カストディ設計**: 委託業者が徴収した資金は、Summit Gateのスマートコントラクトを通じて「直接」各山小屋のウォレットへ分配される。
3. **運営の資金非接触原則（法的安全性の確保と自治体対応）**: 運営側は、委託業者や自治体から日本円やJPYCを一時的にも預からない。
   - **直接発行（Direct Issuance）スキームの採用**: 自治体が「暗号資産を直接保有できない」制約を回避するため、自治体はJPYC社の銀行口座へ日本円を振り込み、受取先として「Summit Gateの分配コントラクト」を直接指定する。これにより、自治体はJPYCを1秒も保有することなく、既存の予算執行（日本円振込）のみでシステムを稼働させる。
   - **法的な解釈**: 自治体側は「環境保全協力金の分配事務」という目的のための「日本円の支出」として処理し、その後の分配はスマートコントラクトという「中立な自動計算機」が執行する形を取る。
   - **リスク排除**: 運営側が「資金移動業」や「カストディ業」に該当するリスクを完全に排除する。
4. **JPYC公式償還スキームの利用**: 銀行振込（日本円への現金化）は、JPYC公式の「償還（Redemption）」サービス（JPYC EX等）を利用する。

### C. 実現戦略
1. **実証実験スキーム**: 条例変更ではなく、期間限定の「実証実験」として行政・組合と連携。
2. **限定展開**: 管理が容易な富士宮ルートに限定して開始。

## 6. 成功指標
- 山小屋における現金管理工数の削減。
- 「タダ乗り」ゼロの実現。
- JPYCによる決済・精算の完結率。
