# プロジェクト意図定義書 - Summit Gate

## 1. やりたいこと (Vision / Purpose)
*   **環境保全金の事前徴収によるトイレの「ウォークスルー化」**:
    *   登山者が小銭を取り出すことなく、通過するだけでトイレを利用できる世界を実現する。
*   **それによる「3つの解放」**:
    *   **1. 山小屋の解放（三重苦の解消）**:
        *   **物理的重労働**: 富士宮ルート（5.5万人）のわずか2割が利用するだけで、**1つの山小屋あたり約160kg**（330万円相当）もの硬貨が発生。「米袋5つ分」以上の重量を人力で運ぶ苦行から解放される。
        *   **金銭的損失**: 銀行入金手数料による利益圧迫をなくす。
        *   **人的リソースの浪費**: トイレ番を廃止し、人手不足を解消する。
    *   **2. 登山者の解放**: 小銭を用意・管理するストレスをゼロにする。
    *   **3. 環境の解放**: 利用実績に基づく公正な資金循環で、持続可能な山岳環境を維持する。

## 2. 何をするか (Action / Solution)
*   **全体像**: 入山料徴収に「相乗り」して環境保全金を集め、トイレ利用数（AIカウント）に応じて山小屋へ公正に分配する。
*   **具体的な実装ステップ**:
    *   **1. 集める（Proxy Collection）**:
        *   入山ゲートにて、入山料（4000円）に「環境保全金（2000円）」を上乗せして一括代理徴収してもらう。「トイレパス」とは呼ばず、あくまで「環境保全」名目とすることで、閉鎖時のクレームリスクを回避する。
        *   **資金フロー（Direct Issuance）**:
            *   徴収した現金は、運営を経由せず**委託業者から直接JPYC社へ振込**。
            *   JPYC社は同額のJPYCを**スマートコントラクトへ直接発行**する。
    *   **2. 測る（AI Counting）**:
        *   各山小屋のトイレ入口にエッジAIカメラ（reCamera）を設置し、プライバシーを侵害することなく「利用数」だけを自動カウントする。
    *   **3. 配る（Fair Distribution）**:
        *   カウントデータに基づき、プールされた資金から各山小屋が整備費として**「Withdraw（引き出し）」**できるブロックチェーン基盤を提供する。
        *   **※Pull型（引き出し）にする理由**:
            1.  **システム保全**: 自動送金（Push）によるシステム停止リスクを回避するため。
            2.  **経理制御**: 山小屋側が資金移動のタイミングを管理できるようにするため。
*   **通過カウントの自動化**:
    *   カメラ映像から「人」を検知し、入室者数を自動カウントする（映像は保存せず、数値データのみ送信）。
*   **ブロックチェーン決済基盤の構築**:
    *   カウントデータに基づき、ステーブルコイン（JPYC）を用いて山小屋へ報酬を**引き出せる（Withdraw）**システムを構築する。
    *   **※JPYCを採用する理由**:
        1.  **法的安全性（資金移動業の回避）**:
            *   運営が日本円を預かると「資金移動業」の登録が必要になるが、本スキームでは委託業者からJPYC社を経由して**スマートコントラクトへ直接JPYCを発行（Direct Issuance）**させる。
            *   これにより、運営は資金の「受取・保管・送金」を一切行わず、単なる「技術提供者」として振る舞えるため、法規制を完全にクリアできる。
        2.  **プログラマブルな分配**: スマートコントラクトにより、AIのカウントデータに基づいた即時かつ透明性の高い分配ロジックを実装できる。
        3.  **親和性**: 既存の金融システム（銀行振込）とWeb3（ウォレット）の架け橋として機能し、将来的なB2B決済への拡張性が高い。

## 3. 解決できること (Merit / Impact)
*   **「小銭の重量」と「手数料」の完全撤廃**:
    *   山から硬貨を降ろす物理コストと、銀行に支払う手数料コストを同時になくす。
*   **「Web3の壁」を意識させないUX**:
    *   **ガスレス（手数料無料）**: 日本円への換金や業者への支払いは、運営がガス代を肩代わりするため、山小屋側は暗号資産（AVAX）を購入・管理する必要が一切ない。
    *   **請求書払い対応**: 受け取ったJPYCをそのまま業者の請求書払いに充当できるため、換金の手間すら省ける。
*   **「トイレ番」の人件費削減**:
    *   常駐スタッフが不要になり、他の業務（接客や安全管理）にリソースを回せるようになる。

## 4. 導入時の問題点と対策 (Feasibility / Issues)
*   **通信環境の不安定さ**:
    *   **対策**: Store-and-Forward（データを蓄積して後で送る）方式を採用し、一時的な圏外やWi-Fi断に対応。Starlinkの活用も推奨。
*   **プライバシー懸念**:
    *   **対策**: エッジ処理により映像そのものは即時破棄し、クラウドには「日時・人数」のみを送信することで匿名性を担保する。
*   **導入コストへの懸念**:
    *   **対策**: 1台あたり約2.2万円（フルセット）の安価な汎用機材（reCamera）を選定し、導入ハードルを下げる。

## 5. 将来的にどうしたいか (Future Roadmap)
*   **完全自動化（自律分散）**:
    *   **スマートコントラクトとオフチェーンシステム**を連携させ、人間による確認作業をなくし、システムが自律的に実績検証・送金を行う仕組みへ移行する（Phase 2）。
*   **B2B経済圏の構築**:
    *   山小屋が受け取ったJPYCを日本円に換金せずに、そのまま業者（ブルドーザー輸送、物資調達）への支払いに回す循環モデルを作る。
*   **「山岳ドミノ戦略」によるキャッシュレス圏の拡大**:
    *   **Phase 1 (山小屋)**: まずは山小屋B2B決済でJPYCを浸透させる。
    *   **Phase 2 (周辺施設)**: 下山後の麓の店舗や入浴施設へ導入を広げ、経済圏を拡張する。
    *   **Phase 3 (登山客)**: 最終的には、インバウンド需要の増加を見据え、登山客が「小銭なし」で山小屋での物販・飲食購入まで完結できるキャッシュレスリゾート化を構想している。
*   **全ルート展開**:
    *   まずは独立性の高い「富士宮ルート」でPoCを成功させ、その実績を元に富士山全4ルートへ展開する。
